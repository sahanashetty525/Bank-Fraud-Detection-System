Bank Fraud Detection System

Design & Analysis of Algorithms – Mini Project

Online banking transactions are increasing rapidly, which also increases the risk of fraud. Manual fraud detection becomes difficult due to the large volume of transaction data.

Problem Statement

To detect suspicious bank transactions by analyzing transaction patterns, money transfer paths, and abnormal transaction spikes.

Objectives

  • Detect repeated suspicious transactions
  • Trace unusual money transfer routes
  • Identify abnormal transaction amounts
  • Apply efficient DAA algorithms

Dataset Used

We use the PaySim synthetic financial dataset which simulates realistic banking transactions. It contains transaction type, step, sender, receiver, amount, and fraud label.

Stakeholders

  • Banks
  • Customers
  • Fraud Detection Teams
  • Regulatory Authorities

Proposed Solution

The proposed system uses efficient Design and Analysis of Algorithms (DAA) techniques to automatically identify suspicious bank transactions in a structured and reliable manner.

  • Data Collection: Transaction details are retrieved from the bank database.
  • Pattern Detection: Rabin–Karp algorithm identifies suspicious sequences.
  • Network Mapping: Accounts as nodes, transactions as edges.
  • Path Tracing: Dijkstra’s algorithm tracks unusual routes.
  • Data Sorting: Merge Sort detects abnormal transaction spikes.
  • Risk Flagging: Suspicious activities are marked for verification.

Algorithm Analysis

Algorithm Purpose Time Complexity Space Complexity
Rabin–Karp Detect repeated or suspicious transaction patterns O(n + m) O(1)
Dijkstra’s Algorithm Track unusual money transfer routes O(E log V) O(V + E)
Merge Sort Organize transactions to detect volume spikes O(n log n) O(n)

These algorithms were selected for their efficiency, scalability, and suitability for handling large financial datasets.
